## Sub Directory Definitions and Rules
1. **core** -- This is the main directory where the user does the planning for the project. All the files in the directory are only read-only for agents. The agents should read the files to stay up to date on the broard vision and objectives of the project. 
2. **prd** -- Project Requirement Documents
3. **implementation** -- Implementation tracking documents. Agents should:
   - Read task specs before starting work
   - Update checkboxes in tasks.md when completing tasks
   - Add session entries to progress.md after each work session
   - Never modify the PRD reference links or task numbering
