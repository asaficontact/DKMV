# Record Architecture Decisions

## Status

Accepted

## Context and Problem Statement

We need to record the architectural decisions made on this project. Which format and process should we use?

## Considered Options

- [MADR](https://adr.github.io/madr/) — Markdown Any Decision Records
- Informal notes in a shared document
- No formal record

## Decision Outcome

Chosen option: "MADR 4.0", because it provides a lightweight, version-controlled format that lives alongside the code.

### Consequences

- Good: Decisions are discoverable and searchable in the repo.
- Good: New contributors can understand the rationale behind past choices.
- Neutral: Requires discipline to create ADRs for significant decisions.
